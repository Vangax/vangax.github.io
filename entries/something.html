research system

neuro reactor: a neural state decoding and visualization system for eeg-based motor imagery

brain–computer interfaces · eeg signal processing · neural representation learning · real-time neural systems

1. original motivation

the initial inspiration behind this project came from speculative ideas about brain-driven experiences, where neural signals could theoretically be translated into interactive perceptual outputs. concepts like “braindance” in science fiction imagine a direct mapping between neural activity and experiential playback.

while those ideas remain largely speculative, they raised a scientific question that is actively studied in neuroscience and machine learning:

can neural activity recorded from the brain be decoded into structured, interpretable representations?

rather than attempting unrealistic goals such as thought reading, the goal of this project was to build a practical brain–computer interface prototype capable of decoding and modeling neural signals produced during motor imagery tasks.

the project therefore evolved into a complete system for decoding eeg signals, learning neural embeddings, and visualizing dynamic neural state behavior in real time.

2. research domain

the project lies at the intersection of several research fields:

  • brain–computer interfaces (bci)
  • machine learning for neural signals
  • eeg signal processing
  • neural representation learning
  • interactive real-time systems

brain–computer interfaces aim to translate patterns of neural activity into machine-interpretable signals. one common paradigm used in bci research is motor imagery, where participants imagine performing specific body movements.

imagining movement produces measurable changes in oscillatory activity in the sensorimotor cortex. these patterns can be detected in eeg recordings and classified using machine learning.

3. dataset

the system uses the widely studied bci competition iv dataset 2a, a benchmark dataset for motor imagery classification research.

  • subjects: 9
  • eeg channels: 22 electrodes
  • sampling rate: 250 hz

EEG Channels

each subject performs trials corresponding to four motor imagery tasks:

  • left hand imagery
  • right hand imagery
  • feet imagery
  • tongue imagery

during each trial the subject imagines performing the movement while eeg signals are recorded. the dataset provides carefully controlled experimental trials and has become a standard benchmark for evaluating bci decoding algorithms.

4. signal processing pipeline

raw eeg signals cannot be directly used by machine learning models because they are noisy, high dimensional, and contain irrelevant frequency components.

therefore the first stage of the system focuses on signal preprocessing and feature extraction.

band-pass filtering

motor imagery activity is primarily expressed within the mu and beta frequency bands. to isolate relevant oscillatory patterns, the eeg signals were decomposed using a filter bank of band-pass filters.

frequency bands included:

  • 8–12 hz
  • 12–16 hz
  • 16–20 hz
  • 20–24 hz
  • 24–30 hz

this multi-band decomposition enables the system to capture discriminative neural patterns across several spectral regions.

epoch segmentation

continuous eeg signals were segmented into epochs corresponding to motor imagery trials. each epoch represents the neural activity during a specific imagined movement.

5. feature extraction using fbcsp

the primary feature extraction method used in this project was filter bank common spatial patterns (fbcsp).

common spatial patterns is a classical algorithm used in motor imagery bci research. it identifies spatial filters that maximize variance differences between classes.

intuitively, the algorithm discovers linear combinations of eeg electrodes that best separate different mental tasks.

fbcsp extends this idea by applying csp to multiple frequency bands, capturing both spatial and spectral information simultaneously.

this approach is well suited for motor imagery decoding because imagined movements produce localized oscillatory power changes in specific cortical regions.

6. feature vector construction

after applying spatial filters to each frequency band, the variance of the filtered signals was computed to produce discriminative features.

these features were concatenated across bands to form a compact representation.

each trial therefore produced a feature vector of approximately 30 dimensions, summarizing the neural signal in a structured format that could be used as input to machine learning models.

7. neural embedding model

instead of using classical classifiers such as support vector machines or linear discriminant analysis, this project introduced a neural embedding model.

the goal was not only classification but also learning a latent representation of neural activity patterns.

model architecture:

  • input: 30-dimensional fbcsp feature vector
  • layer 1: linear layer (30 → 256)
  • activation: relu
  • dropout regularization
  • layer 2: linear layer (256 → 128)
  • embedding representation: 128 dimensions
  • classifier layer: 128 → 4 outputs

softmax activation produces probabilities for the four motor imagery classes.

8. neural embedding representation

the embedding layer creates a latent representation of neural activity.

instead of directly mapping signals to labels, the model learns a neural state space.

within this space:

  • similar mental states produce nearby embeddings
  • different motor imagery tasks form separable clusters
  • neural dynamics can be analyzed geometrically

this allows the system to study representation structure rather than performing pure classification.

9. model training

the model was trained using supervised learning with cross-entropy loss.

evaluation metrics included:

  • training accuracy
  • test accuracy
  • cross-validation performance

single-subject classification accuracy reached approximately 80% or higher, which is considered strong performance for motor imagery eeg decoding.

cross-subject performance was lower due to inter-subject variability, a well-known challenge in bci research.

10. continuous neural state modeling

traditional bci pipelines treat each trial independently. this project explored a different idea:

represent neural decoding as a continuous evolving state.

instead of only predicting class labels, the system derived several metrics from model outputs.

these included:

  • prediction confidence
  • entropy (uncertainty)
  • volatility of class probabilities
  • focus index
  • state stability over time

these metrics allow the system to interpret neural decoding as a dynamic process rather than a static classification task.

11. real-time neural streaming

a backend server was developed to simulate real-time neural streaming.

the server performs:

  • eeg trial loading
  • feature extraction
  • model inference
  • neural metric computation
  • websocket data streaming

example streamed data structure: 

{
 intensity: 0.83,
 class_probs: [0.1,0.8,0.05,0.05],
 dominant_class: 1,
 entropy: 0.34,
 volatility: 0.12,
 focus: 0.64
}

this produces a live neural state stream.

12. interactive visualization interface

the frontend interface was built using react and three.js.

the interface functions as a neural monitoring console displaying:

  • 3d neural core visualization
  • probability bars for mental states
  • entropy and volatility indicators
  • timeline of state transitions
  • neural drift radar plots
  • confidence waveforms

the interface updates continuously as neural state data arrives.

13. system architecture

the overall system pipeline:

eeg dataset → signal processing → fbcsp features → neural embedding model → neural metrics → websocket streaming → interactive visualization

14. project significance

although the system is a prototype, it demonstrates several concepts relevant to active research areas:

  • continuous neural state modeling
  • embedding-based neural representation learning
  • real-time brain-computer interface visualization
  • uncertainty-aware neural decoding

15. final outcome

the final result is an end-to-end neural decoding system capable of:

  • decoding motor imagery eeg signals
  • learning neural state embeddings
  • tracking neural state dynamics
  • streaming neural metrics in real time
  • visualizing neural activity interactively

the project represents an integrated exploration of brain–computer interface design, neural signal processing, and machine learning-based neural representation modeling.
