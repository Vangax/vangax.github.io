1. where i've been

last month i went quiet. no posts, no updates. i was working on something i couldn't really talk about yet: braincomputer interface authentication research, specifically using P300 event-related potentials as a biometric signal. the work is ongoing, it's under proper supervision, and i'll write about the actual research properly when i'm further along. but while i was neck-deep in EEG data every day, i built a tool to help me understand what i was looking at. that tool is what this post is about.

if you want the short version: i built an interactive explorer for the BigP3BCI dataset that lets you load a subject, watch the scalp topography animate across the epoch window, and see the P300 waveform live at any electrode you click. it took a while to get right because the dataset has some quirks that aren't obvious until you've debugged them for several hours.

the P300 is a positive voltage deflection in the EEG that appears around 230 to 500 milliseconds after a stimulus the subject is paying attention to. it is not generated by the eyes or the muscles. it comes from the brain's context updating process: the moment when a relevant event gets integrated into working memory. you can read it off the scalp with electrodes.

2. what the P300 actually is

before the tool makes sense it helps to understand what you're looking at. the P300 is one of the most studied ERP components in cognitive neuroscience. it shows up reliably when someone is attending to a rare or meaningful stimulus embedded in a stream of irrelevant ones. this is called the oddball paradigm. you show someone a rapid sequence of letters flashing on a screen. most are not the target. occasionally one is. the brain generates a large positive voltage peak roughly 300 milliseconds after the target flash. that is the P300.

the signal is maximal at parietal electrodes, especially Pz, the midline electrode at the back of the head. the temporo-parietal junction is thought to be the main generator. the earlier frontal variant (P3a) reflects involuntary attention to novelty. the later parietal one (P3b) reflects deliberate context updating. for BCI purposes what matters is the parietal one because it is the largest and most consistent signal to decode.

the BigP3BCI dataset is a large collection of EEG recordings from subjects performing a P300 speller task: a matrix of letters flashes in rows and columns and the subject attends to the one they want to select. the dataset has multiple studies with multiple subjects and sessions. study A alone has 19 session folders. getting from raw EDF file to a clean ERP with the right preprocessing took more debugging than i expected.

3. the preprocessing rabbit hole

the first version of this tool loaded fine but showed nothing. zero target trials, zero standard trials. the ERP plot was empty. the status bar said "loaded" but there was no data.

the problem turned out to be three separate things layered on top of each other, and each one was subtle enough that i didn't catch it immediately.

the first was event detection. my original code was calling mne.events_from_annotations() which looks for events stored as annotations in the EDF header. BigP3BCI doesn't do that. it stores stimulus timing in two dedicated analog channels: StimulusBegin which transitions from 0 to 1 at every flash onset and StimulusType which is 1 during target flashes and 0 during non-targets. the right approach is to reload the original EDF specifically to access those channels after preprocessing has stripped them out then find the 0 to 1 transitions in StimulusBegin and read StimulusType at each onset sample to get the labels. this is exactly how the reference preprocessing pipeline for this dataset works and i only found it by reading that code carefully.

the second was the artifact rejection threshold. i had set it at 150 µV which is standard for clean 32-channel EEG setups. BigP3BCI files have 114 channels including EOG, EMG and miscellaneous non-EEG channels that naturally have much higher voltage swings. with those channels present every epoch was failing the threshold check and getting dropped. the correct threshold after picking only the EEG channels is 100 µV which is what the reference pipeline uses.

the third was channel naming. BigP3BCI names its EEG channels with an EEG_ prefix so the electrode you know as Pz is stored as EEG_Pz and Cz is stored as EEG_Cz. my lookup tables all used the short names so nothing matched. the fix was to strip the prefix after loading so the rest of the pipeline sees clean standard 10-20 names.

the filter settings also needed changing. i was using 0.5 to 20 Hz which is reasonable for many EEG applications. the P300 has essentially all its energy below 12 Hz so the correct bandpass for this dataset is 1 to 12 Hz plus a 50 Hz notch filter for European power line noise. the tighter filter also helped with the artifact rejection by removing high-frequency muscle contamination before thresholding.

4. what the explorer does

once the preprocessing pipeline was correct the visualization came together quickly. the explorer is a PyQt5 desktop application with two main panels.

scalp topography

a 32-electrode head map that updates in real time as you drag the time slider. the color field is interpolated from electrode amplitudes using cubic griddata interpolation with a circular head mask. red is positive voltage, blue is negative. click any electrode to select it.

ERP waveform

time-locked average across all target trials and all standard trials for the selected electrode. shows the oddball waveform, the standard waveform, and optionally the difference wave. the P300 peak is automatically detected and annotated with latency and amplitude.

animated forward pass

a play button sweeps the time marker from -200 to 800 ms while the topo map updates every 60ms. you can watch the visual components arrive early over occipital electrodes and the P300 build up parietally around 230 to 350 ms.

neuroscience annotations

each electrode region has a brief annotation explaining what you would expect to see there and why. parietal electrodes explain P3b and the context updating literature. frontal ones explain P3a. occipital ones explain why the P300 is weak there.

subject selector

automatically discovers subject folders from the study directory. switch subjects without restarting. the worker thread loads and preprocesses in the background so the UI stays responsive.

peak statistics

per-electrode peak latency and amplitude in the post-200ms window, plus trial counts. useful for quickly checking whether a subject has a clean P300 or whether you should skip them for analysis.

5. the pipeline, step by step

step 1
load EDF and pick EEG channels

read the raw EDF with MNE. channels prefixed EEG_ are kept and renamed to standard 10-20 labels. everything else is dropped. BigP3BCI files have 114 channels total so this step reduces the data significantly.

step 2
apply montage

strip the EEG_ prefix temporarily to match against MNE's standard_1020 montage, set electrode positions, then rename back. this gives us the 3D electrode coordinates needed for topographic interpolation.

step 3
filter and re-reference

bandpass 1 to 12 Hz using a FIR firwin design. notch at 50 Hz. then common average reference (subtract the mean of all electrodes from each electrode at each timepoint). this removes noise shared across all channels equally.

step 4
extract events from stimulus channels

reload the original EDF to access StimulusBegin and StimulusType. find 0 to 1 transitions in StimulusBegin. label each onset as target (1) or non-target (0) by reading StimulusType at that sample. a typical session has around 840 total flashes: roughly 70 targets and 770 non-targets.

step 5
epoch and reject artifacts

cut the continuous signal into -200 to 800 ms windows around each event onset. apply baseline correction using the -200 to 0 ms pre-stimulus window. reject any epoch where a channel exceeds 100 µV. average the surviving epochs separately for targets and non-targets to get the ERP.

6. what the data actually shows

the first subject i loaded (SE001, session A_01) showed a peak at 230 ms at Pz. that is earlier than the 300 to 400 ms you would expect from a slow classical oddball paradigm. the BigP3BCI reference preprocessing code notes this explicitly: the P300 window for this dataset is 150 to 350 ms rather than the typical 250 to 500 ms because the speller task uses a faster stimulus presentation rate. faster presentation compresses the latency. 230 ms is real and expected for this setup.

the topographic map at 230 ms shows a clear posterior positive field concentrated over parietal and centro-parietal electrodes with the maximum at Pz. the frontal electrodes are weakly positive or neutral. occipital electrodes show the residual of the early visual P1 and N170 components that have already passed. this is what a clean P300 looks like.

a note on the dataset. BigP3BCI study A is a P300 speller dataset recorded at 256 Hz with a 9-by-8 stimulus grid. the target probability is roughly 1 in 9 or about 11%. this low probability is what makes the P300 large: the rarer the target the bigger the brain's response. the 68 target trials and 751 non-target trials in the first session are in the right ratio.

7. why i built this

honestly the immediate reason was practical. when you're doing BCI research and you get a new session file you want to quickly check whether the data is usable before running any classification pipeline on it. is there a visible P300? is the peak at a sensible latency? how many trials survived artifact rejection? these are things you can answer by eye in about ten seconds with a good visualizer. without one you're writing one-off matplotlib scripts every time.

the deeper reason is that i think visual tools are underrated in signal processing research. you can have a pipeline that runs without errors and produces numbers and still be doing something wrong. looking at the actual waveforms and the topographic distribution is the fastest way to catch preprocessing mistakes. the empty ERP plot that showed nothing was what told me my event detection was wrong. no amount of print statements would have been as immediate.

the authentication research this was built for is something i'll write about properly later. the short version is that P300 latency and amplitude vary between individuals in ways that are stable over time and hard to spoof, which makes them interesting as a passive biometric. you can't fake your P300 the way you can fake a fingerprint or a password. that property is what makes the signal worth studying for security applications.

the weirdest part of working with EEG data is that you're looking at voltage differences on the order of microvolts recorded through bone and scalp from neurons firing in the brain. the P300 you see in the averaged waveform is maybe 8 to 15 µV above baseline. thermal noise in the amplifier is on the same order of magnitude. the signal is real only because you're averaging hundreds of trials and the noise cancels out while the stimulus-locked response doesn't.

8. in action

the full explorer window: scalp topography (left) and ERP waveform at Pz for SE001. the P300 peak is visible at approximately 230 ms in the target (purple) waveform.

the tool works. it takes a raw EDF from BigP3BCI study A, runs the full preprocessing pipeline in a background thread and shows you the P300 in about 90 seconds. i'll keep using it as the authentication research continues and will probably extend it to compare across subjects once i have more sessions processed.

if you're working with the same dataset and hit the same preprocessing issues i did the code is available and the main things to know are: use the StimulusBegin and StimulusType channels not annotations, pick EEG_ channels by name prefix not by type, use 1 to 12 Hz with a 50 Hz notch and 100 µV rejection threshold, and strip the EEG_ prefix before matching channel names against any standard lookup table. that would have saved me several hours.
